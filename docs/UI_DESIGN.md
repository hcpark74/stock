## 8. 주의사항

1. **실시간 데이터 미연동**: 현재 화면은 하드코딩된 목업 데이터. 실제 운영 시 WebSocket 또는 REST 폴링으로 교체 필요.
2. **모바일 미지원**: 아크 캔버스와 테이블이 좁은 화면에 최적화되어 있지 않음.
3. **브라우저 보안**: `data/db/trading.db`는 Python 서버 없이 브라우저에서 직접 접근 불가.
4. **NTP 경고 임계값**: ±200ms 초과 시 상태 바 점이 Amber로 전환 (PRD §4 기준).

## 7. 2026-07-01 UI 동작 업데이트

### 하단 파이프라인

- 하단 파이프라인은 단순 `position_status`가 아니라 `/api/status`의 `pipeline_stage`, `pipeline_failed`를 우선 사용한다.
- `pipeline_stage` 인덱스는 다음 의미를 가진다.
  - `0`: F1 스캔
  - `1`: F1 완료
  - `2`: F2 잠금 또는 F3 진입 시도
  - `3`: F4 Step Trailing
  - `4`: F5 타임아웃 또는 종료 단계
- `pipeline_failed=true`이면 현재 단계는 실패 색상으로 표시한다.
- 예: F3 미체결 실패 후 `position_status=IDLE`로 돌아가도 하단은 `F1 완료 > F2 완료 > F3 실패` 흐름을 유지한다.

### F1 후보 패널

- 후보 목록은 원 랭킹 순서보다 통과 가능성을 우선한다.
- 정렬 우선순위는 선정 후보, 갭 통과, 예상체결 갭 통과, 랭킹 갭 통과, 예상체결 유효성, 거래대금 순이다.
- `HIGH_GAP_VI_UNKNOWN`은 UI에서 `VI미확인`으로 표시한다.
- 스냅샷이 존재하면 F1 로그 상태가 `IDLE`이어도 후보 요약을 표시한다.

### 이벤트 로그

- F3 진입 과정의 세부 이벤트를 표시한다.
  - `F3_RECHECK`
  - `F3_ENTRY_BLOCKED`
  - `ENTRY_PRE_ORDER_WAIT`
  - `ENTRY_ORDER_SENT`
  - `ENTRY_FILL_POLL_TIMEOUT`
  - `ENTRY_CANCEL_SENT`
  - `ENTRY_RETRY_START`
  - `ENTRY_RETRY_SKIPPED`
  - `ENTRY_FAIL`
- 주문번호, 주문가, 주문수량, 재시도 횟수, 체결조회 요약은 이벤트 상세 텍스트에 함께 노출한다.

## 8. 2026-07-02 메뉴 기준 업데이트

### 자산 메뉴

- 목적은 계좌의 현재 스냅샷을 빠르게 확인하는 것이다.
- 주요 지표는 총평가금액, 예수금, 주문가능금액, 보유종목 수, 주식평가금액, 평가손익이다.
- 원천 데이터는 `/api/assets`의 KIS 잔고 조회 결과이다.
- KIS 조회 성공 결과는 `asset_snapshots` DB 이력으로 저장하며, 최신 메모리 캐시가 없을 때는 마지막 저장 스냅샷과 `captured_at` 시각을 보조 표시한다.
- `주문가능금액`은 주문 화면에서도 참조될 수 있지만, 데이터 소유 기준은 자산 메뉴에 둔다.
- KIS 잔고 응답에 주문가능 현금 필드가 없으면 예수금을 fallback으로 표시한다. 종목별 정확한 주문 가능 여부는 주문 실행 전 매수가능조회 결과를 따른다.

### 주문 메뉴

- 목적은 주문 실행과 처리 결과를 확인하는 것이다.
- 화면 제목은 `주문/체결 내역`을 우선한다. 사이드 메뉴 라벨은 공간상 `주문`을 유지할 수 있다.
- 주요 지표는 오늘 주문 수, 체결 수, 미체결 수, 취소/실패 수이다.
- 본문은 주문 타임라인 테이블을 중심으로 구성한다.
- 테이블 컬럼은 주문번호, 종목, 매수/매도, 주문수량, 주문가격, 체결수량, 상태, 주문 단계, 주문시각/체결시각을 기본으로 한다.
- 데이터는 `/api/orders`에서 조회한다. `/api/stream`의 로그 이벤트가 들어오면 즉시 재조회하고, 5초 폴링을 백업으로 사용한다.
- 주문가능금액을 표시할 때는 큰 요약 카드가 아니라 작은 참조 배지나 보조 텍스트로 노출한다.

### 용어

- 메인 대시보드의 `가용주문` 문구는 `주문가능금액` 또는 축약형 `주문가능`으로 표기한다.
- `자산 메뉴에서 상세` 문구는 유지해 주문가능금액의 원천이 자산 데이터임을 알려준다.

### 오늘 화면

- 오늘 화면은 운영 상태 확인에 집중한다.
- F1 후보 전체 테이블은 오늘 화면에서 노출하지 않고 선정 메뉴에서 확인한다.
- 오늘 화면의 F1 영역은 진행 단계, 최종 후보, 최종 갭, 예상체결 대금, 후보 전체 보기 액션만 표시한다.
- 보유 중에는 `보유 후 가격흐름` 차트를 표시한다.
  - 서버의 최근 tick ring buffer와 WebSocket/SSE tick을 함께 사용해 선형 흐름을 그린다.
  - 진입가, 최고가, Hard Stop 또는 Trail Stop 기준선을 함께 표시한다.
  - 포지션이 없거나 청산된 상태에서는 대기/청산 상태 텍스트를 표시한다.
- 자산 메뉴의 주문가능금액에는 `주문가능 현금` 또는 `예수금 기준` 출처를 보조 텍스트로 표시한다.
- 주문/체결 내역은 실패/취소 시 `error_msg` 또는 `error_code`를 사유 열에 표시한다.
