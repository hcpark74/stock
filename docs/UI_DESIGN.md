# 화면 설계 문서 — 갭업 트레이딩 봇 모니터링 UI

> **버전**: 1.0  
> **최종 수정**: 2026-06-23  
> **대상**: 운영자 단독 사용 (모바일 미대응, 데스크톱 브라우저 전용)

---

## 1. 설계 원칙

| 원칙 | 내용 |
|------|------|
| 한 눈 확인 | 아침 장전 5초 안에 오늘 상태 파악 가능 |
| 숫자 중심 | 가격·수량·% 모두 모노스페이스 폰트로 정렬 |
| 상태 색상 | 수익=Cyan(#22D3EE), 손실=Red(#FB4B4B), 주요이벤트=Amber(#F5A623) |
| 시간 중심 | 80분 세션(08:40→10:00)을 아크 게이지로 시각화 |

## 4. 화면 목록

### 4-1. 오늘 (Today Dashboard)

**목적**: 현재 포지션 상태를 즉시 파악

**레이아웃**:
```
┌──────────────────────────────────────────────┐
│ NAV: ⬆ GAP TRADER v1.0          [오늘|이력|통계] │
├──────────────────────────────────────────────┤
│ STATUS: KIS정상 · WS연결 · NTP+143ms · 09:15 KST │
├─────────────────────────┬────────────────────┤
│                         │ 이벤트 로그         │
│  [세션 아크 캔버스]       │ 09:15 틱 수신      │
│  08:40 ──●──── 10:00    │ 09:08 1차 익절     │
│  [진입] [익절] [현재▶]   │ 08:59 진입 체결    │
│                         │ 08:58 F2 종목 잠금  │
│  상태: 보유중  005930    │ 08:40 F1 필터      │
│  삼성전자                │ 08:30 NTP/토큰     │
│ ┌────────────────────┐  │                    │
│ │진입가  현재가  손익  │  │                    │
│ │75,800  77,200 +1.84%│  │                    │
│ │최고가  잔여  투입금   │  │                    │
│ │77,750   62주  8.11M │  │                    │
│ │[1차익절✓][BEP●][TR대기]│ │                   │
│ └────────────────────┘  │                    │
└─────────────────────────┴────────────────────┘
```

**핵심 컴포넌트**:

| 컴포넌트 | 설명 |
|----------|------|
| 세션 아크 | 08:40~10:00 80분을 반원 게이지로 표현. Amber 채움 = 경과 시간 |
| 이벤트 마커 | 진입(🟡)·1차익절(🔵)·현재(●)를 아크 위에 표시 |
| 상태 워터마크 | "보유중" 130px Amber 반투명 텍스트를 배경에 배치 (심미적 리스크) |
| 포지션 카드 | 6개 지표 3열 그리드 + 플래그 행 |
| 이벤트 로그 | 시각 오름차순 8건, 레벨별 색상 구분 |

**상태별 화면 변화**:

| 상태 | 워터마크 | 포지션 카드 | 아크 색상 |
|------|---------|------------|----------|
| IDLE | 매수대기 | 빈 상태 | 트랙만 표시 |
| ENTERING | 진입중 | 주문 진행 표시 | Amber 소량 |
| HOLDING | 보유중 | 수치 표시 | Amber 채움 |
| CLOSED | 청산완료 | 최종 손익 표시 | 전체 채움 |

## 5. 공통 컴포넌트

### 네비게이션 바 (고정)
- 브랜드: `⬆ GAP TRADER v1.0` (Amber, 모노스페이스)
- 탭 3개: 오늘 / 이력 / 통계 (pill 형태)

### 상태 바 (Nav 하단)
- KIS API 상태 (초록 점)
- WebSocket 연결 상태 (초록 점)
- NTP 오차 (±200ms 이내=초록, 초과=노랑)
- KIS_MODE 표시 (PAPER/REAL)
- 실시간 KST 시계 (1초 갱신)

---

## 8. 주의사항

1. **실시간 데이터 미연동**: 현재 화면은 하드코딩된 목업 데이터. 실제 운영 시 WebSocket 또는 REST 폴링으로 교체 필요.
2. **모바일 미지원**: 아크 캔버스와 테이블이 좁은 화면에 최적화되어 있지 않음.
3. **브라우저 보안**: `data/db/trading.db`는 Python 서버 없이 브라우저에서 직접 접근 불가.
4. **NTP 경고 임계값**: ±200ms 초과 시 상태 바 점이 Amber로 전환 (PRD §4 기준).
